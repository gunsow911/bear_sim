# 対策コマンド（施策）を5本立てに拡張する

日付: 2026-07-01

## 背景と意図

現状の地区指定コマンド（対策コマンド）は `広域草刈り` と `電気柵` の2つだけ。
山口県「第二種特定鳥獣（ツキノワグマ）管理計画（第5期）」の施策5本柱
（個体群管理／被害防止対策／生息環境管理／錯誤捕獲の防止／普及啓発）を下敷きに、
**プレイヤーが毎ターン能動的に地区へ指示できる施策**を5つに拡張する。

設計の軸：

- **能動的に指示できるもの**＝地区を選んで指示ポイントで打つ戦術コマンドに限る。
  「状況に応じて選ぶ／備える」性格のもの（学習放獣・普及啓発）は議題／イベント層へ回す。
- **管理リソースを増やさない**。プレイヤーが見張るメーターは既存の
  `指示ポイント` と `不満度` の2つだけ。捕獲枠などの新カウンタは追加しない。
- **数値マスキングは維持**。UI表示文（flavor / effectLabel）に数値は出さない。

## 5施策と計画の対応

| # | 施策 | ActionKind | 計画の本柱 | 役割 |
|---|---|---|---|---|
| 1 | 緩衝帯の刈り払い（現「広域草刈り」） | `mowing`（既存） | 生息環境管理 | 経路を断つ・予防 |
| 2 | 誘引物の除去（カキ・生ゴミ） | `attractant-removal`（新） | 被害防止（重点） | 餌/動機を削る・持続 |
| 3 | 電気柵の設置 | `electric-fence`（既存） | 被害防止（防護） | 出没を1回止める盾 |
| 4 | 箱わなによる捕獲 | `box-trap`（新） | 個体群管理 | 里山の源を減らす予防的除去 |
| 5 | 緊急銃猟 | `emergency-shooting`（新） | 個体群管理／人身被害防止 | 市街決壊への緊急対応 |

補足：

- 4（箱わな＝里山・予防）と 5（緊急銃猟＝市街・緊急）で除去のステージを分け、冗長を避ける。
- 緊急銃猟は 2024 年の鳥獣保護管理法改正で創設された新制度で、本計画（令和4年策定）より
  新しいが、計画の「個体群管理／緊急時対応」の精神に合致するため採用する。

## 各施策の詳細（レバーは調整前提の初期値）

数式・状態は `src/engine/model.ts`（`RiskModelParams`）と `src/engine/turn.ts`、
`src/types/index.ts`（`DistrictState`）に対応する。レバー値は仮で、チューニングで動かす。

### 1. 緩衝帯の刈り払い `mowing`（既存・変更なし）

- **効果**：対象地区の流入（山林直接流入=第1項・隣接移動=第2項）を
  `mowingBlockTurns` の間 `mowingInfluxCutRate` ぶんカット。
- **コスト**：指示 1。数ターンで失効（継続整備が必要）。
- **表示**：既存の flavor / effectLabel を踏襲。
- 用語ツールチップ：`緩衝帯整備`。

### 2. 誘引物の除去 `attractant-removal`（新）

- **レバー（里山・市街の両方に効く）**：柿・生ゴミは里山だけでなく市街地の出没
  （空き家の柿・屋外の生ゴミ）も引き寄せるため、両方の上昇圧を持続的に抑制する。
  - `DistrictState.intervention.satoyama` を負の値へ駆動（`satoyamaRise` の加算項＝里山抑制）。
  - `DistrictState.intervention.urban` を 1 未満へ駆動（`urbanRise` の乗算係数＝市街決壊の抑制）。
  現状 intervention は「駆動源待ちで中立保留」（`2026-07-01-remove-neglect-and-cleanup-design.md`）
  なので、本施策が里山・市街それぞれの再駆動源になる。
- **持続**：即効ではなく、数ターン持続する介入として置く
  （実装案：`interventionTurns` を新設し、有効な間だけ里山の負加算・市街の <1 乗算を適用。
  失効で satoyama=0 / urban=1＝中立へ戻す）。恒久にはしない（放置すれば戻る）。
- **コスト**：指示 1。効きは緩やかだが持続。低密度・早期の地区でじわ効く。
- **表示例**（数値なし）：
  - name: `誘引物の除去`
  - flavor: 「放置された柿や栗、屋外の生ゴミを片付ける誘引物除去を地域ぐるみで進めます。
    里山でも市街でも、クマが人里へ通う“動機”そのものを、しばらくの間そぎ続けます。」
  - effectLabel: 「この地区の里山・市街の出没圧をしばらく抑え続ける」
  - realTerms: `['誘引物除去']`
- **議題との整理**：既存の議題 `attractant-removal`（全地区 里山 -6）は概念が重複する。
  地区コマンド化に伴い、議題側は撤去するか別名（全域一斉版）に整理する（下記「議題・イベント層」参照）。

### 3. 電気柵の設置 `electric-fence`（既存・小改修の余地）

- **効果**：`electricFenceTurns` の間、里山出没を1回だけ無効化（発揮で即失効）。
- **改修案（任意）**：計画が農地・養蜂場の防護を主眼にするため、
  里山だけでなく市街出没も1回無効化する対象に含めてよい（`fence-block` を urban にも適用）。
  スコープを絞るなら現状（里山のみ）維持でも可。**初期は現状維持**とし、改修は別途判断。
- **コスト**：指示 1。1回発揮で失効。
- **箱わなとの優先順位（確定）**：同地区で電気柵と箱わなが両方有効なとき、里山出没1回に対しては
  **箱わな（捕獲）を優先し、電気柵は消費せず温存する**。捕獲は「不満0＋源の恒久ダウン」で
  より価値が高く、罠にかかった個体は柵に到達しないという物理にも合う。結果、
  `罠で1頭捕らえ源を減らす → 消費後は残った柵が次の出没を防ぐ` の重ね置き相乗効果になる
  （二重消費・併置禁止は不採用）。

### 4. 箱わなによる捕獲 `box-trap`（新／案A＝待ち伏せ→捕獲で源を恒久に下げる）

唯一「守り（出没を捕獲に置換）」と「源を減らす（以後その地区が楽になる）」を兼ねる施策。
電気柵・草刈り・誘引物除去のどれとも役割が被らない。

- **動作**：
  1. **設置**（指示 2）。設置しただけでは即効果なし。有効 N ターンの待ち伏せ状態に入る
     （実装：`DistrictState.trapTurns` を新設）。
  2. 有効な間、その地区で**里山出没が起きるはずのタイミングで、出没を「捕獲」に置き換える**。
     住民から見れば「出た」ではなく「罠にかかった」→ **不満増はゼロ**（電気柵に似た防御）。
     この捕獲成立で待ち伏せは消費して終了（電気柵と同様に発揮で失効）。
  3. **捕獲が成立すると、その地区の山林直接流入（第1項）を一段階、恒久に下げる**。
     実装：`DistrictState.forestInfluxFactor`（初期 1.0）を新設し、`satoyamaRise` の第1項
     （山林直接流入）にこの係数を掛ける。以後その地区が楽になり、隣接へ配る第2項も細るため
     波及効果がある。
- **複数回捕獲のスタック（乗算＋下限クランプ）**：捕獲成功のたびに `forestInfluxFactor` を
  **×0.7**（例）する。乗算なので回を重ねるほど絶対効果が逓減し（1頭目が最も効く）、
  終盤の雪だるまを自然に抑える。**下限 0.25〜0.3 でクランプ**し、どれだけ捕っても源はゼロに
  ならない（＝クマは減らせても消せない、計画のトーンに合う）。加算（−0.3 ずつ）は 3〜4 回で
  0＝地区の完全無害化になり終盤の張り合いが消えるため不採用。
  - 例：1.00 → 0.70 → 0.49 → 0.34 → 0.30（下限）
- **適用範囲（確定）**：第1項（山林直接流入）は `mountainAdjacent` の地区にしか存在しないため、
  `forestInfluxFactor` による恒久の源ダウンは**山林隣接の地区でのみ効く**（徳地・阿東など＝本来の主戦場）。
  非隣接の地区（源は隣接流入=第2項）では、箱わなは**その回の出没を捕獲する防御は効くが恒久の源ダウンはない**。
  「森の源がない地区では捕っても湧き元は減らない（隣から来るだけ）」という現実に合致する。
  → 市街寄り／非隣接の地区に置いても効果が薄いのは仕様。
- **空振り**：有効 N ターン中に里山出没が起きなければ捕獲不成立＝源ダウンなし。
  高圧地区に先回りして仕掛ける読み合いが生まれる。
- **コスト**：**指示 2**。捕獲枠カウンタは設けない（指示消費と1地区縛りが自然な制限）。
- **バランス懸念**：恒久ダウンは終盤ほど有利になりうる。16 ターンなので下げ幅は控えめ
  （×0.7 程度）にし、下記の錯誤捕獲イベントをブレーキにする。
- **世論の代償（メーター化しない）**：多用を内部フラグ（隠し）で数え、
  一定回数を超えると**錯誤捕獲・放獣を巡るランダムイベント**（不満増・人身事故など）を
  誘発しやすくする。UI にカウンタは出さない。計画の「年間上限135頭」の精神は
  この乱獲イベントでフレーバー的に効かせる。
- **表示例**（数値なし）：
  - name: `箱わなによる捕獲`
  - flavor: 「排除地域に箱わなを仕掛け、里へ通う個体を待ち受けます。かかれば人里に出る前に
    捕らえられ、以後その地区に降りてくる圧そのものが和らぎます。ただし人手と手間がかかります。」
  - effectLabel: 「この地区に来た個体を捕らえ、以後の里山の出没圧を和らげる（人手が要る）」
  - realTerms: `['個体数管理']`（用語辞書に応じて調整）

### 5. 緊急銃猟 `emergency-shooting`（新）

- **発動条件（ゲート）**：対象地区の `urbanEncounterRate` が閾値以上
  （＝市街決壊が起きている／寸前）のときのみ選択可。平時はグレーアウト。
  → 乱用不可の“最後の防衛線”。`canAfford` に条件判定を足すか、UI で発動可否を出す。
- **効果**：対象地区の `urbanEncounterRate` を**即時に大きく叩き落とす**
  （市街に出た個体を緊急排除）。必要なら `satoyamaEncounterRate` も併せて下げる。
- **代償**：発動で**不満度が少し上がる**（発砲・安全確保・住民動揺）。
  加えて事故のランダムイベント可能性（低確率）。指示コストは重め（例：2）。
- **表示例**（数値なし）：
  - name: `緊急銃猟`
  - flavor: 「市街地に居座る個体へ、市町の判断で緊急の銃猟を実施します。決壊した市街の
    危険を即座に断ち切りますが、発砲は住民を動揺させ、事故の恐れもあります。」
  - effectLabel: 「市街に出た個体を緊急排除する（住民は動揺する）」
  - realTerms: `['緊急銃猟']`（用語辞書に追加）
- 用語辞書 `緊急銃猟`：「市街地等でクマ等が緊急の危険を及ぼす際、市町村の判断で銃による
  捕獲を実施できる制度（2024年鳥獣保護管理法改正で創設）。」

## 議題・イベント層へ回すもの

- **学習放獣**（既存議題 `aversive-release`）：錯誤捕獲個体の放獣。状況依存のため議題／イベントで存続。
- **普及啓発**（既存議題 `awareness`）：住民心理＝不満度（全地区共通）に効く。地区戦術ではないので議題で存続。
- **既存議題 `attractant-removal`**：施策2と重複。全域一斉版として残すか撤去するかを実装時に判断。
- **重い分岐**：捕獲の乱用時の錯誤捕獲・人身事故・世論反発、緊急銃猟の発砲事故 → ランダムイベントの選択肢。

## 非目標（YAGNI）

- 捕獲枠・予算などの新規管理リソースは追加しない。
- ゾーニング（排除/緩衝/保護）による施策の可否切り替えは今回入れない
  （将来拡張の余地として残す）。
- 電気柵の市街対応拡張は初期スコープ外（現状維持、別途判断）。

## 確定事項（このセッションで決定）

- 5施策の顔ぶれ：刈り払い／誘引物除去／電気柵／箱わな捕獲／緊急銃猟。
- 学習放獣・普及啓発は議題／イベント層。捕獲枠などの新リソースは追加しない。
- 誘引物除去は里山・市街の両方に効く（satoyama 負加算＋urban <1 乗算）。
- 箱わなは案A（待ち伏せ→捕獲で源を恒久ダウン）。コスト**指示2**。
- 箱わなの源ダウンは乗算スタック（×0.7）＋下限クランプ（0.25〜0.3）。第1項＝山林隣接地区限定。
- 同地区で箱わな・電気柵が競合したら**箱わな優先・電気柵温存**。

## 未確定・実装時に詰める点

- 施策2の持続ターン数と、里山負加算量・市街乗算係数（<1）の具体値。
- 施策4の待ち伏せターン数 `trapTurns`、`forestInfluxFactor` の刻み（×0.7 の可否）と下限値。
- 施策5の発動閾値（市街遭遇率いくつ以上で可）・遭遇率の低下量・不満加算量・指示コスト。
- 箱わな乱用の内部カウンタ閾値とイベント誘発ロジック（別スペック化も可）。
- 電気柵の市街対応拡張（初期は現状維持）。
