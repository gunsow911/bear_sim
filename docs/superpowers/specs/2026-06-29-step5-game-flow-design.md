# Step 5 設計：ゲームフローの確定（議題・対策・遭遇）と啓蒙演出

## 背景・目的
エンジン（数理モデル）と地図はできている。本ステップでは **1ターンの流れ** を確定させ、
プレイヤーが「議題 → 対策 → 遭遇」を自然に回せるようにする。Plague Inc 風の
ダッシュボード体験を保ちつつ、熊害の **啓蒙**（専門用語の理解）を促す。

確定済みの方針（ユーザー合意）:
- 議題フェーズ：**突発イベントはモーダル通知**、**アジェンダは画面内カードで3枚から1枚選択**
- 遭遇フェーズ：**地図演出（出没地区を強調）＋ 上部サマリー**
- **フェーズ名（議題/対策/遭遇）は開発上の呼称。ゲーム画面には出さない**
- ターン表記：**「第X週 / 14」**
- 突発イベント頻度：**時々（毎ターン約35%で抽選）**
- アジェンダ/イベント効果：**シンプルな状態変更を数種**（後から拡張）
- **専門用語はマウスオーバーで解説**（啓蒙）

## ターン進行モデル
内部フェーズは保持するが、画面には「週」と文脈的な操作のみ見せる。進行は右上の
**単一の文脈ボタン**で行う（フェーズ名は出さない）。

```
週の開始
  ├ 突発イベント（約35%で発生）→ EventModal で通知（なければスキップ）
  └ アジェンダ3枚を上部に提示 → 1枚選択で可決（無料・純バフ）→ 対策操作が解放
対策
  ├ 地区を選び対策コマンドを指示（予算・指示P消費）
  └ 右上ボタン「クマの行動へ」
遭遇
  ├ 遭遇を解決し、地図で出没地区を強調＋上部に結果サマリー、遭遇率の色を更新
  └ 右上ボタン「次の週へ」
判定：不満度100 → ゲームオーバー / 第14週まで完走 → 防衛成功
```

HUD：`第X週 / 14`・予算・指示P・不満度・活発度。**フェーズ名ラベルは撤去**。
右上ボタンの文言は内部フェーズで切替（アジェンダ未選択時は無効/非表示）。

## データ＆エンジン拡張
数理は `RiskModel` のまま据え置き。議題・イベントは**モデルに依らない状態変更**として実装。

- `src/data/agendas.ts`：アジェンダ定義（id / 名前 / 説明 / 効果）。
- `src/data/events.ts`：突発イベント定義（id / 名前 / 説明 / 発生確率 / 効果）。
- `src/engine/agenda.ts`（純関数）:
  - `rollEvent(rng): EventDef | null` … 週開始の抽選（合計約35%）
  - `applyEvent(game, event): GameState`
  - `pickAgendas(rng): AgendaDef[]` … プールから3枚抽選
  - `applyAgenda(game, agenda): GameState`
- 効果の種類（当面）：`budget` / `instructionPoints` / `activeness` / `dissatisfaction` の増減、
  および「このターン限定の対策効果倍率」フラグ（store が対策適用時に参照）。

ストア（`gameStore`）拡張:
- 状態：`currentEvent`（モーダル表示用）/ `agendaChoices`（3枚）/ `selectedAgenda` /
  `turnModifiers`（当ターン限定バフ）
- アクション：`startTurn()`（イベント抽選＋アジェンダ抽選）/ `chooseAgenda(id)` /
  `dismissEvent()`。`advancePhase` を上記フローに沿って改修。

## UI 構成（Framer Motion で軽い演出）
- `EventModal`：突発イベント通知（中央モーダル、確認で閉じる）。
- `AgendaCards`：上部に3枚のカード。選択で可決し対策が解放。
- `EncounterReveal`：遭遇解決時に地図上の出没地区をハイライト/🐻アイコン、上部に結果サマリー。
  既存の `EventLog` バナーを置き換え/統合。
- `GlossaryTerm`：専門用語をラップし、ホバーで解説ツールチップを表示する小コンポーネント。
- HUD 改修：フェーズ名撤去、週表記、文脈ボタン。

## 専門用語ツールチップ（啓蒙）
**熊害に関する現実の知識**に関心を持ってもらうことが目的。ツールチップは
**現実の対応・事象・問題を表す用語に限定**して付ける。

- 対象（ツールチップを付ける）＝**現実の熊害用語のみ**：
  **学習放獣**・誘引物（放置果樹・生ゴミ）除去・侵入防止柵（電気柵）・緩衝帯整備・
  ゾーニング管理・個体数管理（管理捕獲）・出没マップ（YPくまっぷ）・ドングリ凶作と出没の関係 など
- 対象外（ツールチップを付けない）＝**ゲーム内部の用語**：活発度・里山遭遇率・市街遭遇率・
  決壊係数 等。ゲームの仕組み説明は今回は行わない。
- `src/data/glossary.ts`：現実用語 → 解説のマップ。各エントリは短い説明（必要なら出典/補足）。
- `<GlossaryTerm term="学習放獣">学習放獣</GlossaryTerm>` でホバー時にツールチップ表示。
- **イベントの説明文・アジェンダ**は現実の対応/事象/問題を題材にし、その中の現実用語をラップする。

## イベント・アジェンダの題材（現実準拠）
ゲーム数値を動かすだけでなく、文面・題材を現実の熊害トピックに寄せる。
- 突発イベント：ドングリ大凶作（堅果不作と出没増の関係）／市街地出没の通報多発／
  冬眠前の荒食い期 など、現実に起きる事象。
- アジェンダ：学習放獣・誘引物除去キャンペーン・緩衝帯の草刈り・注意喚起/啓発・人員動員 など、
  現実の自治体対応。効果はシンプルな状態変更にしつつ、文面と用語で現実に接続する。

## 初期コンテンツ（シンプル・後で拡張）
アジェンダ（純バフ・無料）:
- 緊急予算（予算+10万）／注意喚起（活発度-10）／人員動員（指示P+1）／
  広域草刈り強化（このターン草刈り効果UP）／啓発活動（不満度-5）／早期警戒（活発度上昇を一時抑制）
- 学習放獣（現実の手法）：全県の活発度を一段抑制（人里回避の学習を表現）。用語ツールチップで現実の手法を解説
  ※ v1 ではアジェンダは全て**全体/即時の純バフ**に統一し、地区指定（対象選択UI）は持たせない。

突発イベント（約35%で1件抽選）:
- ドングリ大凶作（活発度+40）／出没多発（ランダム地区の里山遭遇率+）／
  寒波（活発度+）／ボランティア来援（予算少額+）

## 既存との接続・スコープ外
- 既存の対策コマンド（草刈り/クリーン作戦/電気柵）・遭遇解決・勝敗判定はそのまま活用。
- 季節変動の本格モデル化（暦に連動した活発度推移）は **スコープ外**（イベントで擬似的に表現）。
- 用語集・アジェンダ/イベントは最小セットから開始し、データ追加で拡張する。
